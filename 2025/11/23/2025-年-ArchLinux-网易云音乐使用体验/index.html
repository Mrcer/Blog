2025 年 ArchLinux 网易云音乐使用体验

目前我的工位电脑主力系统是 ArchLinux。作为一个耳机常年长在脑袋上的人、网易云重度依赖者，安装好系统的第一件事当然是安装网易云音乐。但这过程并不平坦，接下来我分享一下这一年来我安装不同版本网易云音乐的尝试和使用体验……

下面所有过程都在写作时重新尝试并审视了，但因为 ArchLinux 环境复杂，我不保证在其他环境能完全复现。

网页版

维度 评分 评价
安装 ★★★★★ 不用安装
功能 ★★★☆☆ 只听歌也够了
体验 ★★★☆☆ 中规中矩的 Web 应用，细节欠打磨
性能 ★★★★★ 即用即开毫无压力
总体 ★★★★☆ 新系统的首选

以前实验课用 Linux 就一直在 Web 版上听歌。听收藏、日推没什么问题，但如果要使用别的功能就不太方便了，比如心动模式、一起听是没有的，歌词显示仅限直接 div 铺开的字体，没有置顶显示或滚动歌词。歌单显示也很糟糕，最多显示1000条但没有翻页功能，也不支持搜索。除此之外，可能因为安全原因，Web 版的登录更容易过期。

Deepin 版

维度 评分 评价
安装 ★★★★☆ 用 archlinux 的都会使用 aur 源吧
功能 ★★★★☆ 比 Web 版多了歌词，基本对齐旧版 wyy 了
体验 ★★★☆☆ 很多API过期了导致网络错误，但还能用
性能 ★★★★★ 秒开，还得是旧版 PC 端
总体 ★★★★☆ 体验比 Web 版好，但 bug 更多

因为想和朋友一起听歌，但 Web 版不支持，所以我又一次踏上了折腾的路……

我首先考虑的是遵循 arch wiki 的建议，安装了 netease-cloud-music 的 AUR 包。这本质是和 Deepin 联合适配 Linux 的原生应用，理应是最合适的……吗？

虽然很感谢社区和网易开发者的支持，但这个项目确实半截入土了，AUR 显示 Last Updated: 2024-03-18 10:12 (UTC)。这带来很多问题，最严重的大概是 API 过期：登录的时候不支持验证码，400 错误也不知道是不是输错密码，总之绕道微信登录了；刚登录时歌单加载是有问题的，需要重启应用才能正常使用；个性推荐界面是不可用的，大概率是 API 变动了。不过这一切熟悉后，基本算是一个旧版的网易云音乐，PC 端该有的功能都有了。除了要容忍一些 bug，比如歌词无法拖拽，其他体验都不错。

然而不支持最新功能，也就是不支持心动模式和一起听，那我还要继续试……

Wine (Bottles) 运行 PC 版

维度 评分 评价
安装 ★★☆☆☆ Wine 依赖地狱
功能 ★★★★★ 官方支持的客户端，功能很全
体验 ★★★★☆ 小小瑕疵，总体能接受
性能 ★★★☆☆ 新版PC端本来就慢，而且要 Wine 转译
总体 ★★★★★ 很艰难才能跑起来，但至少有新功能了

既然常规客户端缺乏维护，那只能邪修转译别的客户端了，比较接近的 PC 端自然是首选。参考网上现有的转译方案，我选择了 Bottles 前端的 Wine。Bottles 提供了好看的前端，以及多种 wine 后端和依赖的管理功能，但网络问题和各种小 bug 令人有些烦躁。 不过感谢开发者，在编写该博客时，Bottles 有了很大改进，尽管网络问题还是很糟糕——这里建议大家用命令行启动 Bottles，能更方便地查看日志，至少知道他卡在哪了。

接下来最地狱的是安装依赖。不同 wine 后端、不同功能启用、不同的依赖（列表里有好几十个）安装得到的运行效果都是不同的，你只能通过日志不断调整运行方式，相当绝望。可以确定的是，你至少要安装 Webview2，然后想办法解决字体问题。字体问题很头疼，尤其是当你看到满屏幕方块时。我尝试过安装字体依赖，但似乎没用，而且很奇怪的是感觉他用的是 arch 系统上的字体。我通过在设置中不断更换尝试不同字体，总算让 UI 显示正常了。

总之，如果你排除万难启动了他，并且解决了字体问题，那么这一版方案应该是最好的了！能使用最新更新的功能，心动模式、一起听、歌词都没问题。不过还是有不少瑕疵，比如不要尝试启用音乐桌面，歌词边缘有锯齿，任务栏显示拉胯，关机前要先关应用不然会卡住关机过程。

Waydroid 运行安卓版

维度 评分 评价
安装 ★★★☆☆ 安装时需要一些配置和尝试
功能 ★★★★☆ 官方支持的客户端，但歌词没了
体验 ★★☆☆☆ 手机界面不适配桌面
性能 ★★★☆☆ 首次启动要先开安卓系统，对于音乐播放器很糟糕了
总体 ★★★☆☆ 除非你已经有了安卓模拟器，不然不推荐

在我尝试完这个方法后，我写下了这篇博客，因为实在很想吐槽。之前用 wine 装了好几次网易云，每次 bug 都不太一样，最后一次是字体问题，但也凑合用了。后来朋友指点用安卓模拟器，包能解决字体问题，而且安卓本来就基于 Linux，兼容性和性能都更好。想来也是，说干就干吧。

按照网上教程，Waydroid 是 linux 上比较推荐的开源安卓模拟器。安装过程遇到了个坑，提示内核缺少 binder、veth 之类的模块。这很奇怪，因为 arch wiki 上都没提到这回事。一通问 AI，发现是系统更新没重启，这下知道 windows 上更新并重启的良苦用心了。系统启动后还要装网易云音乐，遗憾的是网易云音乐只编译了 arm 架构，因此还需要配置转译层。按照 wiki 引导尝试了适用于 amd 的 libndk，应用不能打开。社区说这个包兼容性不好，所以换成了 libhoudini，这下能打开了。

界面是 pad 适配的，但显示在桌面大小还是走样了。安卓模拟比 wine 兼容性好很多，功能和字体都正常，毕竟环境隔离程度更高。然而很糟糕的是，界面是无法缩放的，就算打开了多窗口模式也无法正常使用，界面更是格格不入，也不会有歌词显示。

Splayer

维度 评分 评价
安装 ★★★★☆ AUR 助手一键安装
功能 ★★★★☆ 比官方还多一些功能，但也少一些功能
体验 ★★★★★ 感谢开发者，界面真的很好看
性能 ★★★★☆ 需要开屏等待，但能接受
总体 ★★★★★ 没有特别需求的话，体验已经非常好了

写这篇博客时，发现还有一种方式没有尝试，那就是 Arch Wiki 上推荐的第三方客户端 SPlayer。简单试了下，发现界面做得非常流畅，果然还是原生好。不过遗憾的是没有做一起听，没有满足我的核心需求，而且我对第三方使用 API 的合法性感到担忧，所以还是继续选用 wine 的方案。

总结

总的来说，推荐只有听歌需求的同学使用 SPlayer，使用体验相当优秀。如果要使用官方客户端的话，使用 wine 运行 PC 端是最好的。
