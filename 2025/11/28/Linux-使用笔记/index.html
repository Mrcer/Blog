结缘 Linux

I use Arch, btw.

虽然看起来很搞，但现在我确实用 archlinux 作为主力机，而且并非为了装（应该）。且听我慢慢道来。

从 Ubuntu to Go 开始

如果没记错，第一次见到 Linux 桌面版应该是在高中信息竞赛时，我还在无聊地被老师远程控制桌面，而我朋友竟然可以继续打代码。

这一下子吸引了我注意。细看两眼，这系统、这代码、这调试如此与众不同！那时，我还在 windows 上用 dev-c++ 编写运行调试，而朋友是在用 ubuntu 上用 vim、g++、gdb 做题。这给了我不小震撼，我还记得他当时应该在尝试用字节形式实现大数计算——现在我也会了，甚至可能会写 SIMD 加速哈哈。不过，当时最羡慕的还是可以在老师讲课时敲代码。于是我向他取经，这才知道他在 U 盘上装了 Ubuntu Live。和正常操作系统不同，Live 系统开机后不会修改本地磁盘数据，一旦关机系统就会被“还原”。这通常是用来安装新系统的，如果用作日常使用就要解决数据持久化的问题。我在学习他制作了这样的启动 U 盘后，就曾多次因为碰到 U 盘而导致系统直接卡死，导致代码丢失。后来，我学会了在 U 盘上划分一个持久化分区，从而实现重要数据的持久化。

除了丢失数据的可能外，每次全新系统的网络配置、环境配置也是稍微麻烦的一点。所幸信息学竞赛不需要太复杂的环境，只要安装 vim、g++、gdb 即可使用；
网络配置对于当时的我来说不太容易，那时我还没理解五层网络模型这些知识，只会按朋友指导配好IP、Mask、DNS这些，不过用多几次也就学会了。

说了这么多缺点，其实用 U 盘上的 Live 系统还是有一些优点的。首先他安装非常简单，只需要找个烧录软件把镜像写进 U 盘就行了，不需要配置引导、不用担心破坏任何数据（对于空 U 盘），甚至不用“安装”。
其次，能够带着 U 盘到处跑，随时随地都能“夺舍”一台电脑，这本身就很酷！最后，Live 系统不用考虑 U 盘性能问题，毕竟启动介质是只读的，大大减轻了 IO 压力。

这是我第一次使用 Linux 系统，也是我第一次接触 javac 以外的各种命令。g++、gdb 这些底层工具的使用让我提前接触到了系统相关的知识，为我后面计科学习种下了个种子。

尝试更多使用方式

上大学后，我陆续尝试了更多使用 Linux 的方式。

首先是完整的系统。Live 系统面对轻量任务时虽然还能胜任，但每次还原系统终究还是太不方便了。不过，我那时还不会拆机加硬盘，更别提分区装双系统——不过我倒是在之前装机时拜托过亲戚装 Windows-Ubuntu 双系统。此外，我也是一个胆子比较小的人，担心在自己电脑上折腾出事。于是我第一次装的系统是在 U 盘上进行的，其实也很简单，就是电脑上插了两个 U 盘，一个 Live 一个目标 U 盘，启动 Live 系统后安装到目标 U 盘就完事了。

此外，我还了解到了 Termux，以及类似的 Aid Learning。他们吸引我的点和 Ubuntu to Go 类似，那就是随时随地用自己的系统写代码！可能这看起来很好笑，尤其是当我上课掏出手机支架、移动键盘，用 vim 在小小屏幕上写代码时，或者是我尝试在 aarch64 上尝试编译安装 numpy、scipy 这些时。不过那个时候我已经有点喜欢上折腾了，只要随便有点理由，比如我不想带几斤重的游戏本，或者是我想用手机跑代码，我就会做。现在看来，这些也不是全无收获的，我确实在上面写过上百行代码吧，现在我也会用 termux 跑些 traceroute 之类的软件。

当然，由于课程需要，我日常使用的电脑，也就是我的游戏本，也需要一个 Linux 环境。我不记得谁先谁后了，依稀记得自己在上面装过 wsl、虚拟机、双系统，但最后稳定在了 wsl 上。wsl 确实是和 windows 共存的最佳选项，如果你只是想拥有 linux 环境的话。

KDE 让我投奔了 ArchLinux

用过 Ubuntu 的都知道，他的界面和 Windows 是有不少区别的，而且当时整体 ui 设计偏拟物一点。所以当我第一次看到装了 kde plasma 桌面的 archlinux，看到他熟悉的桌面布局、融为一体的窗口状态栏、简洁清新的扁平设计、好看的桌面壁纸（？），我就被吸引过去了。进一步了解，发现 ArchLinux 提供了相当完善的文档，很适合我了解 linux 相关知识，于是就决定投奔 ArchLinux 了。

一开始 ArchLinux 也是一样装在 U 盘上的，只是我换了一个比较高速的 U 盘。第一次装 ArchLinux 时跟随 wiki 安装，连桌面环境都没安装，进系统后在 tty 上折腾了半天网络配置，还为了简洁只装了 plasma 基本包，导致缺这少那的。这是我第一次见识到 archlinux 相对 ubuntu 的折腾，不过我说服了自己，更何况之前已经有不少折腾经验了。后来 archlinux 越用越顺手——命令行体验完爆没有配置过的 windows——所以我经常在主力机上启动它办公。

后来，我买了小主机，往上面装了 Windows-ArchLinux 双系统，从此工位上基本都启动 ArchLinux 了。现在看来，ArchLinux（或者任何 Linux 发行版）在有一定计算机知识储备的情况下，用多了之后也不会比 Windows 难用，甚至开发体验还要更加顺手。当然，偶尔你会不得不切换到 Windows，比如需要用 office 全家桶，比如一些专有软件。

然而，即便在日常使用中，偶尔你也会需要学习探索，解决一些 linux 上的小问题……

字体

字体配置在很多 linux 发行版中都很重要，但在 archlinux 中尤为如此……

  1. 基础知识
    • fontconfig 是底层字体配置，所有应用都会用它
    • plasma 也提供了字体配置，但那只适用于部分应用
    • 关于字体设计的常识：全角/半角，衬线/无衬线/等宽
  2. 中文显示方块：安装中文字体
  3. 关字窄窄的，门字穿出去：修改 locale
  4. 我输入法呢：大多数时候，装一下 fcitx5 等输入法；但有一些应用有输入法也打不开，比如 wps，那就只能搜索解决了，这也是 linux 不得不品的一环
  5. 安装宋体后好多地方变宋体了：配置正确的后备字体

这时就会发现，很多时候认真阅读文档就能解决这些问题，而且 arch wiki 确实写得很好看……RTFM! (sorry)
